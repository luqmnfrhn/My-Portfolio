### 2. Screenshot carousel

Replace current `smartschool-showcase` block with a CSS-driven auto-slide carousel.

**Behavior:**
- Single mixed carousel — web and mobile screenshots together
- Auto-advances every 4.5s via CSS `animation` + `@keyframes`
- Dot indicators show current slide position; clicking dot jumps to that slide
- Thin orange progress bar animates under active slide (4.5s, resets each slide)
- Pause on hover via ~15 lines of vanilla JS
- To add a screenshot: drop `<img>` inside `.ss-slides`, no other changes needed

**Layout:**
- Container: `16/9` aspect ratio, `border-radius`, `border: 1px solid var(--border)`, `background: var(--surface2)`
- Images: `object-fit: contain` — web fills naturally, mobile letterboxed with surface2 sides
- Progress bar: 3px orange line at bottom of carousel, `width: 0 → 100%` over 4.5s

**Structure:**
```html
<div class="ss-carousel">
  <div class="ss-slides">
    <img src="..." class="ss-slide" alt="...">
    <!-- add more imgs here -->
  </div>
  <div class="ss-progress"><div class="ss-progress-bar"></div></div>
  <div class="ss-dots"><!-- generated by JS --></div>
</div>
```

**Implementation approach:** Pure CSS `@keyframes` for N slides using `animation-delay` per slide. JS only handles: pause on hover, dot generation, dot active state sync.
